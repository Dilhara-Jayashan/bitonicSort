## ðŸ”§ Development Guidelines

### Code Style
- Follow C11 standard
- Use clear, descriptive variable names
- Add comments for complex logic
- Keep functions modular and focused

### Testing
- Test on multiple platforms (macOS, Linux)
- Verify with different dataset sizes
- Ensure compilation with both GCC and Clang
- Check both single and multi-thread/process scenarios

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, etc.)
- Keep the first line under 50 characters
- Add details in the body if needed

Example:
```
Add error handling for file I/O operations

- Check fopen return value
- Handle memory allocation failures
- Add informative error messages
```
